Indieweb Reply Preferences

You can set different URLs to redirect to based on the verb implied through the hijack context — for example, a hijacked reply button will trigger the reply action, hijacked Tweet buttons will trigger the post action.

Placeholders (in the form of {name}) can be inserted into these URLs, which will be replaced with the associated bit of metadata if it’s available, or an empty string otherwise.

{url}
Expands to the (urlencoded) URL of the page you’re posting
{via}
Twitter @name from Tweet buttons
{text}
Pre-filled text from Tweet buttons
{hashtags}
Any pre-filled hashtags from Tweet buttons

Replaces:

  • Tweet Buttons across the web
  • Twitter.com retweet buttons
{url}
Expands to the (urlencoded) URL you’re replying to
{username}
Expands to the @name of the Twitter user you’re replying to

Replaces:

  • Twitter.com reply links
{url}
Expands to the (urlencoded) URL you’re favouriting
{text}
Expands to the text of the tweet you’re replying to
{username}
Expands to the @name of the twitter user you’re favouriting

Replaces:

  • Twitter.com favourite links
  • Facebook Like Buttons across the web
